# Phase 5: Advanced True Multi-Color Printing (AMS/MMU & Tool-Changers)

## Objective
True AMS/MMU multi-color printing (single fused mesh, embedded per-instance color) is explicitly deferred to this phase. Early phases support multi-color output strictly via separate single-color batch files or by instructing users to snap on official LEGO pieces. Phase 5 will introduce deep MMU integration, climaxing with **Full-Spectrum CMYKW Blending** to generate millions of colors.

## Key Deliverables
- **Multi-Material 3MF Export (Discrete Per-Instance Mosaics):** A single 3MF file with embedded per-instance/per-vertex color assignments. *Important distinction:* Unlike CMYKW dithering, discrete per-instance color (where each Mosaic stud is a large contiguous color block assigned to an AMS slot) has a vastly superior waste profile since there are no microscopic fraction-of-a-mm layer purges. This forms a **third, fully viable assembly path** for Phase 1 Mosaics beyond buying real bricks or batch-printing, and is fully supported for AMS/MMU.
  - *Multi-Slicer Validation:* Slicers interpret multi-material 3MF dialects inconsistently. An explicit validation step must confirm correct color-slot mapping in at least **Bambu Studio** and **OrcaSlicer/PrusaSlicer** before calling the exporter done.
- **Full-Spectrum CMYKW Blending (Lithophanes Only):** Integrate logic inspired by projects like `3DRev/Primed3D` to calculate exact filament mixing ratios of Cyan, Magenta, Yellow, Key (Black), and White (CMYKW) to dynamically generate millions of true-color pixels via halftone Z-layer dithering (budgeting for 5 filament channels, not 3).
  - *Architectural Scope:* This technique relies on optical blending of Z-layers viewed transversally. Therefore, it is strictly scoped to **Lithophanes**, where light passes *through* the layer stack. It is mathematically incompatible with flat, top-down Mosaics.
  - *Slicer Constraints:* The dithered export requires the user to manually match an exact layer height in their slicer (0.08mm or 0.12mm is proven best). The UI must prominently surface this hard requirement before exporting the 3MF.
  - *Transmission Distance (TD) Calibration:* Implement HueForge's industry standard "Transmission Distance" metric. The platform must include a per-filament TD calibration workflow (print a test strip, visual match, fit a transfer function) so the optical blending math matches the physical reality of the filament.
  - *Purge Waste & Hardware Strategy:* Generating CMYKW on standard single-extruder AMS/MMU systems results in ~97% purge waste (e.g. 34 hours and 437g of filament for a 14g print). The UI must include a strict pre-export estimate of filament waste % and print time delta. This feature should be explicitly targeted at **Tool-Changing Printers** (like Snapmaker/Prusa XL) as the supported path, and marked as highly discouraged/experimental for AMS/MMU setups.
  - *Implementation Strategy:* Since `3DRev/Primed3D` already includes its own in-browser WebGL preview, CMYKW dithering, and 3MF export logic, adopt it as the single unified reference for this feature.
  - *Differentiation:* Emphasize that our unique value proposition against first-party free tools like Bambu's Chroma Canvas is the **studded brick geometry integration** (e.g., generating the lithophane as a mechanically adjacent, friction-fit part to a structural LEGO frame), not just the CMY blending itself.
- **LDraw Multi-Color Mapping:** If the LDraw instructional generator (Phase 4) is utilized in conjunction with full multi-color parts, this phase will implement the canonical numeric color-code mapping back to LDraw's palette.
- **Support Generation / Painting Tools:** Web-based brush tools to explicitly paint multi-color support interfaces onto generated geometries prior to export.

## Patent Freedom-to-Operate (HARD GATE)
- **Do Not Defer:** Patent infringement is about *what* you built, not how you built it. If CMYKW dithering is patent-encumbered, the entire phase architecture is exposed. This cannot be deferred to a post-launch cleanup.
- **Action:** Before starting Phase 5 development, consult an IP attorney to explicitly search the existing diffusion/dithering AM patent family (e.g., US Patent `11,704,858`) and identify the current active patents circulating in hobbyist communities to ensure Freedom-to-Operate for this commercial feature.
